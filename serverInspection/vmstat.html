Linux vmstat 명령어 정리

기본 개념

  • Run-Queue는 실행중 + 대기중 프로세스 전체 포함
  • 스케줄러 입장에서 다음에 실행할 놈을 고르는 것이기 때문

기본 옵션

  • vmstat n1 n2 — n1초마다 갱신, n2번 출력 후 끝
  • -s — 요약 통계
  • -d — 디스크 통계
  • -m — 슬랩 캐시 정보

프로세스 상태 - procs

  • r — 실행 대기 중 프로세스 수
    • Run-Queue
    • running or runable(R 상태)에 해당하는 프로세스 수
    • I/O 대기중인 프로세스는 (D 상태) b 컬럼 반영
  • b — I/O 대기 프로세스 수, 인터럽트 불가능한 수면 상태

메모리 - memory

  • swpd — swap 사용량
  • free — 사용 가능한 여유 물리 메모리
  • buff — 버퍼로 사용중인 메모리
  • cache — 캐시로 사용중인 메모리

스왑 - swap

  • si — swap in(디스크->메모리)
  • so — swap out(메모리->디스크)

디스크 I/O - io

  • bi — block in 블록 읽기
  • bo — block out 블록 쓰기

커널활동 - system

  • in — 초당 발생한 인터럽트 수
  • cs — 초당 발생한 컨텍스트 스위치 수

CPU 사용률 - cpu

  • us — 사용자 영역 CPU 점유율
  • sy — 커널 영역 CPU 점유율
  • id — idle 점유율
  • wa — I/O wait 점유율
  • st — steal
