메모리 페이지 관리 점검 명령어 (Linux)

1) 페이지 사용 현황 확인

/proc/meminfo

cat /proc/meminfo
  • Active / Inactive
  • AnonPages
  • Mapped
  • Dirty
  • Writeback

2) 페이지 교체 및 스왑 활동

vmstat

vmstat 1
  • si → swap in (디스크 → 메모리)
  • so → swap out (메모리 → 디스크)

sar (설치 시)

sar -B 1 5
  • pgpgin/s → 페이지 인
  • pgpgout/s → 페이지 아웃
  • pgfault/s → 페이지 폴트

3) 페이지 폴트 확인

vmstat

vmstat -s | grep fault

sar

sar -B

4) 프로세스별 페이지 사용

pmap

pmap -x PID
  • Address Space 확인
  • Private / Shared 페이지 확인

/proc

cat /proc/PID/smaps

5) 페이지 캐시 확인

cat /proc/meminfo | grep -E 'Cached|Buffers'
  • Cached → 페이지 캐시
  • Buffers → 버퍼 캐시

6) Dirty 페이지 상태

cat /proc/meminfo | grep Dirty
  • Dirty → 아직 디스크에 기록되지 않은 페이지
  • Writeback → 디스크 기록 중

핵심 점검 포인트

항목 확인 내용
Swap 활동 si/so 증가 여부
Page Fault pgfault 급증 여부
Dirty 과도하게 높지 않은지
Inactive 회수 가능한 페이지 여부
