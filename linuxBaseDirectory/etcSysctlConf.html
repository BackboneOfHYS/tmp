Linux /etc/sysctl.conf 정리

기본 개념

  • 커널의 동작 방식을 결정하는 파일
  • 커널 전체 체질을 조절
  • sysctl -a커널 파라미터 출력
  • sysctl -psysctl.conf 내용 즉시반영

주요 역할

  • 네트워크 최적화
  • 메모리 관리
  • 보안 강화
  • 파일시스템
  • 기타 등등

공유메모리

  • 여러 프로세스가 함께 사용하는 메모리 공간
  • kernel.shmmax단일 공유 메모리 세그먼트(덩어리)의 최대 크기
  • kernel.shmall시스템 전체에서 사용가능한 공유메모리 총 페이지 수
  • 4KB대개 메모리 페이지 크기
  • 전체 사용할 공유 메모리 바이트 / 4096 = kernel.shmall로 보통 조절함
  • 질문 -> 공유 메모리를 한 번에 안 주고 단위를 두는가?
  • 답변 -> 최대 할당량 지정, 관리 및 할당에 용이하므로

세마포어

  • 공유자원 접근에 순서제어
  • 세마포어는 공유자원의 접근에 대한 신호등
  • 신호등이 있고, 신호등 그룹이 있고 전체 신호등 개수가 있는 것
  • kernel.sem = SEMMSL SEMMNS SEMOPM SEMMNI세마포어 설정값 형식
  • SEMMSL하나의 세마포어 세트당 가질 수 있는 최대 세마포어 개수
  • SEMMNS시스템 전체에서 쓸 수 있는 총 세마포어 개수
  • SEMOPM한 번의 시스템 호출로 작업할 수 있는 최대 작업 개수
  • SEMMNI시스템 전체에서 만들 수 있는 세마포어 세트의 최대 개수

메시지 큐

  • 프로세스 통신의 메시지 용량 설정
  • kernel.msgmax메시지 한 개 최대 크기
  • kernel.msgmnb메시지 큐 총 크기
  • kernel.msgmni시스템 전체 메시지 큐 최대 개수

코어

  • 스케줄링, 분산처리, 성능제한
  • 세부적인 것은 /sys/devices/system/cpu에서 관리
  • kernel.sched_min_granularity_ns프로세스에게 부여되는 최소 실행시간
  • kernel.sched_latency_ns프로세스가 적어도 한 번은 실행되도록 하는 전체 주기
  • kernel.sched_wakeup_granularity_ns프로세스가 깨어나고 현재 실행중인 프로세스를 밀어내고 코어를 차지할지 결정하는 결정값
  • kernel.numa_balancing구조에서 메모리와 가까운 CPU 코어로 자동 배치
  • kernel.sched_migration_cost-nsCPU코어에서 다른 코어로 프로세스 마이그레이션이 발생할 때 비용 판단 기준시
  • kernel.panicCPU코어 오류나 커널 패닉 발생시 자동 재부팅까지 대기시간
  • kernel.core_pattern프로세스 비정상 종료 시 생성되는 코어 덤프 파일의 경로, 파일명 지정
  • kernel_nmi_watchdog특정 코어가 완전히 멈췄는지 감지하는 기능
  • kernel.perf_event_paranoidCPU코어 성능 카운터를 어디까지 모니터링할 수 있는자
  • kernel.kptr_restrict커널 심볼 주소 노출 제한
