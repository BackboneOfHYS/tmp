Linux /etc/Security/limits.conf 정리

기본 개념

  • 프로세스, 유저의 시스템 자원 접근 한도 조절
  • ulimit -a자신의 ulimit 설정 정도를 볼 수 있음
  • ulimit 적용은 로그인, 로그아웃해야 적용됨

항목별 시그니처

  • as - virtual memory프로세스당 사용 가능한 최대 가상 메모리 크기
  • 설정불가 - pipe sizepipe size, 512bytes
  • nice - scheduling priority프로세스의 스케줄링 우선순위(0~20) 숫자가 낮을 수록 순위 높음
  • locks - fild locks사용자가 걸 수 있는 최대 파일 잠금(Fcntl locks) 개수

개수 관련(단위: 수량)

  • nofile: 열 수 있는 최대 파일 수 (단위: 개)
  • nproc: 실행 가능한 최대 프로세스/스레드 수 (단위: 개)
  • maxlogins: 한 사용자의 최대 동시 로그인 수 (단위: 회)
  • maxsyslogins: 시스템 전체의 최대 로그인 수 (단위: 회)

2. 메모리 및 크기 관련 (단위: KB)

  • memlock: 메모리에 고정(Lock) 가능한 최대 주소 공간 (단위: KB)
  • rss: 최대 상주 세트 크기 (단위: KB)
  • stack: 최대 스택 크기 (단위: KB)
  • data: 프로세스가 사용할 수 있는 최대 데이터 세그먼트 크기 (단위: KB)
  • core: 코어 파일의 최대 크기 (단위: KB)
  • fsize: 생성 가능한 최대 파일 크기 (단위: KB)

3. 시간 및 기타 관련

  • cpu: 최대 CPU 점유 시간 (단위: 분)
  • msgqueue: POSIX 메시지 큐에 할당된 메모리 (단위: Bytes)
  • priority: 사용자 프로세스의 실행 우선순위 (단위: Nice 값, -20~19)
  • rtprio: 최대 실시간 우선순위 (단위: 우선순위 수치)
  • sigpending: 대기 가능한 최대 시그널 수 (단위: 개)
