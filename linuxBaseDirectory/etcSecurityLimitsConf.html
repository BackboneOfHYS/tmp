Linux /etc/Security/limits.conf 정리

기본 개념

  • 프로세스, 유저의 시스템 자원 접근 한도 조절
  • ulimit -a자신의 ulimit 설정 정도를 볼 수 있음
  • ulimit 적용은 로그인, 로그아웃해야 적용됨

항목별 시그니처

  • nofile - open files한 프로세스가 열 수 있는 최대 파일 수
  • nproc - max user processes사용자가 생성할 수 있는 최대 프로세스/스레드 수
  • core - core file size프로그램 비정상 종료 시 생성되는 코어 덤프 크기
  • rss - max memory size프로세스가 점유할 수 있는 최대 물리 메모리 크기
  • stack - stack size프로세스의 최대 스택 영역 크기
  • fsize - file size사용자가 생성할 수 있는 최대 파일 크기
  • cpu - cpu time프로세스가 사용할 수 있는 최대 CPU 시간
  • memlock - max locked memory메모리에 고정(Lock)할 수 있는 최대 주소 공간
  • as - virtual memory프로세스당 사용 가능한 최대 가상 메모리 크기
  • 설정불가 - pipe sizepipe size, 512bytes
  • data - data seg size프로세스의 데이터 세그먼트 최대 크기
  • nice - scheduling priority프로세스의 스케줄링 우선순위(0~20) 숫자가 낮을 수록 순위 높음
  • sigpending - pending signals대체 가능한 최대 시그널 수
  • msgqueue - POSIX message queuesPOSIX 메시지 큐에 사용할 수 있는 최대 바이트 수
  • locks - fild locks사용자가 걸 수 있는 최대 파일 잠금(Fcntl locks) 개수
  • rtprio - real-time priority실시간(Real-time) 스케줄링 우선순위
