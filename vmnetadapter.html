VirtualBox 네트워크 어댑터 모드 간단 정리

기준: “VM↔VM / VM↔호스트 / VM→외부 / 외부→VM”

NAT 가장 간단

  • VM → 외부 인터넷 ⭕
  • 외부 → VM ❌ (포트포워딩 필요)
  • VM끼리 직접 통신 ❌

어댑터 브리지(Bridged) 실제 LAN 참여

  • VM ↔ 외부 ⭕
  • 외부 ↔ VM ⭕
  • VM이 “진짜 PC 한 대”처럼 보임

내부 네트워크(Internal Network) 완전 격리

  • VM ↔ VM ⭕
  • VM ↔ 호스트 ❌
  • VM ↔ 외부 ❌
  • VM들만 연결된 가상 스위치

호스트 전용(Host-only) 관리/접속

  • VM ↔ 호스트 ⭕
  • VM ↔ VM ⭕
  • VM ↔ 외부 ❌ (기본)
  • 호스트에서 VM SSH/관리하기 좋음

일반 드라이버(Generic Driver) 특수 목적

  • 실험/연구용 옵션(UDP 터널, VDE 등)
  • 일반 학습/운영에서는 거의 사용 안 함

NAT 네트워크(NAT Network) 여러 VM + 인터넷

  • VM ↔ VM ⭕
  • VM → 외부 ⭕
  • 외부 → VM ❌ (포트포워딩 필요)
  • DHCP/네트워크 단위로 관리 가능

한 눈에 비교

모드 VM↔VM VM↔호스트 VM→외부 외부→VM
NAT ❌ ❌ ⭕ ❌
브리지 ⭕ ⭕ ⭕ ⭕
내부 네트워크 ⭕ ❌ ❌ ❌
호스트 전용 ⭕ ⭕ ❌ ❌
NAT 네트워크 ⭕ ❌ ⭕ ❌
일반 드라이버 ⚠️ ⚠️ ⚠️ ⚠️

목적별 추천

  • 인터넷만 필요 → NAT
  • 실제 LAN에 참여 (서버처럼 보이게) → 브리지
  • 라우터/VRRP/클러스터 같은 격리 실습 → 내부 네트워크
  • 호스트에서 VM 접속/관리 → 호스트 전용
  • 여러 VM + 인터넷 동시에 → NAT 네트워크
