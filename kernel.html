리눅스 커널 구조

1. 시스템 콜 인터페이스 (System Call Interface)

사용자 프로그램이 커널 기능을 요청하는 통로

  • open()
  • read()
  • write()
  • fork()

2. 프로세스 관리 (Process Management)

  • CPU 스케줄링
  • 프로세스 생성 및 종료
  • 컨텍스트 스위칭
  • 우선순위 관리

3. 메모리 관리 (Memory Management)

  • 가상 메모리
  • 페이지 관리
  • 스왑 처리
  • 메모리 보호

4. 스토리지 스택

사용자 -> VFS -> real FS -> LVM -> 블록 디바이스 -> 물리 디스크

다양한 파일 시스템을 하나의 인터페이스로 통합

  • VFS
  • LVM

5. 디바이스 드라이버 (Device Drivers)

  • 디스크 드라이버
  • 네트워크 카드
  • USB 장치
  • GPU

6. 네트워크 스택 (Network Stack)

  • TCP/IP 처리
  • 소켓 관리
  • 패킷 라우팅
  • 방화벽 처리

구조 계층도

[ User Application ] ↓ [ System Call Interface ] ↓ -------------------------------- | Process Management | | Memory Management | | VFS | | Network Stack | | Device Drivers | -------------------------------- ↓ [ Hardware ]
