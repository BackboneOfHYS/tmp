1. 루트권한을 얻었다고 가정
2. 암호화 방식 추출
3. shadow파일 salt, 해시값 추출 (사용자 중 일부가 취약비밀번호 사용하는 것으로 가정)
4. 사전공격 진행으로 사용자명-비밀번호 쌍 추출

import crypt

hash_value = "$6$abc123$M4Y..."

dictionary = ["123456","password","admin123","qwerty"]

for pwd in dictionary:
    if crypt.crypt(pwd, hash_value) == hash_value:
        print("found:", pwd)

============= 초기 컴퓨터 명령어 삽입 과정 ==

0. os가 없던 시절에는 전면 패널에 버튼으로 관리함
1. 메모리에 명령어 입력 (직접 입력, 펀치카드, 모니터 프로그램(?))
2. PC설정 및 RUN 누름

- 부트로더의 시작이 이런 과정에서 일부 명령어를 미리 저장하고 주소 지정으로 하는 것으로 시작

- batch 처리 등장 (펀치카드를 넣어줌)
- batch 처리에서 환경 세팅을 사용자가 하던걸 resident monitor가 대신해줌
- I/O 컨틀롤러에서 CPU로 인터럽트 신호 보내줌(I/O가 느려서 idle이 생기고 기다렸음)
- I/O 기다리는 시간이 길어서 메모리에 여러 프로그램을 올리는 것으로
- 이를 관리하는 게 os로 이 외에도 cpu, 메모리, I/O, 파일시스템, 프로세스 관리를 함
- 여기에 멀티유저 모드까지 추가

- 현대 OS에서 부팅되고 점검된 후 CPU의 첫 주소는 표준으로 지정되어있다.
- 그리고 이 주소는 reset vector라고 하며 rom주소로 매핑되어잇다.

- 메모리맵과 가상메모리는 다르다. 
- 메모리맵은 cpu가 임의 주소를 보고 어디로(메모리, GPU메모리, 다바이스 등) 송수신할 지 정하는 것
- 메모리맵은 부팅 직후 os가 올라가기 전에 BIOS/UEFI가 정리해서 os에게 알려준다.
- BIOS는 부트로더를 실행하고 부트로더가 os를 메모리에 올림

========================= 공인 IP 서브과정

1. ISP(Internet Service Provider) 모뎀으로부터 WAN선 내려옴
2. 이를 라우터에 연결
3. 라우터가 공인 IP를 가지며 이를 사설IP로 변경
4. 이중화는 WAN선 이중화, 라우터 이중화 등이 있음
5. 클라우드 환경에서 하나의 공인IP에 여러 도메인이 묶여있는 경우는 L7 스위치가 헤더를 보고 도메인을 판별하므로 가능함
