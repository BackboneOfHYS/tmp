Linux free 명령어 정리

기본 개념

  • 시스템이 메모리를 어떻게 사용하는지 출력
  • 리눅스는 남는 공간을 buff/cache로 적극 활용

옵션

  • -hHuman_readable
  • -b, k, m, g단위 고정 Byte, ...
  • -t하단 RAM과 SWAP의 합계(Total)라인을 추가
  • -s [초]지정 초 단위로 반복출력
  • -c [횟수]지정 횟수만큼 출력, -s옵션과 함께 사용
  • -wWide mode. buff와 cache를 분리, 상세 출력
  • -Vfree 명령 버전 정보 확인

일반 메모리 vs 버퍼 vs 캐시

  • 일반 메모리 - 실제 프로세스가 점유
  • 버퍼 -
    • 하드디스크 같은 블록장치에 데이터를 쓰기 전에 잠시 머무는 공간
    • 디스크의 어느 위치에 저장될지 분석하여 데이터 정렬 및 병합
    • 응용 프로그램이 요청한 데이터 단위와 하드웨어가 다루는 데이터 단위가 다른 것을 해결
    • 메타 데이터 저장(권한, 소유자, 생성 시간 등)
    • 저널링으로 안전하게 기록
  • 캐시 -
    • 디스크에서 읽어온 파일 데이터를 메모리에 남겨둔 것
    • 읽기 성능 최적화 - 곧 읽을 것 같은 데이터를 미리 가져옴
    • 쓰기 성능 최적화 - 지연 쓰기, 변경된 내용을 바로 반영하지 않은 더티페이지를 가지고 있음, 커널이 나중에 저장
    • 위 과정으로 프로그램은 바로 다음 과정을 진행할 수 있음
    • 실행 파일 및 라이브러리 공유
  • 버퍼 vs 캐시
    • 캐시는 파일 내용 중심, 논리적인 속도 향상
    • 버퍼는 디스크 블록 중심, 물리적인 효율

장치파일 유형

  • 블록장치파일 - 블록 단위, 임의 접근(random access) 가능
  • 문자장치파일 - 문자 단위, 스트림 형태
  • 네트워크장치파일 - 소캣 단위(?)
