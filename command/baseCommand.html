Linux 기본 명령어

참고사항

  • /dev/null — 들어오는 모든 입력 버림

sudo, su

  • su — 계정 전환하되 환경변수 그대로
  • su - — 타겟 계정의 환경변수로 환경 초기화와 같이 전환
  • sudo su — sudo는 루트권한이 기본으로 들어가므로 root 비밀번호 몰라도 됨
  • sudo — sudo는 root권한만 설정되며 $PATH는 초기화되고 환경변수도 일부 초기화됨
  • sudo -i — root로 로그인한 것과 동일한 환경으로 처리됨
  • sudo -E — 기존 사용자 환경변수를 동일하게 유지하고 싶을 때

권한 설정

  • SUID/SGID — 실행시 파일 소유자/그룹 권한으로 실행, 설정되면 x대신 s, x가 없는데 s있으면 S
  • Sticky bit — 디렉토리에서 삭제 제한, 디렉토리에 w권한있으면 파일삭제 가능(파일 권한과 무관)
  • useradd [계정명] — 유저 생성(리눅스)
  • userdel [계정명] — 유저 삭제(리눅스)
  • groupadd [계정명] — 그룹 생성(리눅스)
    • -g그룹 아이디 지정
  • groupdel [계정명] — 그룹 삭제(리눅스)
  • mkuser home=?, pgrp=?, groups=?, — 유저 생성(유닉스)
  • passwd 계정명 — 유저 비밀번호 설정
    • /etc/passwd에서 보조그룹은 안 보임
  • gpasswd [그룹명]— 그룹 비밀번호 설정
    • -a 유저명유저 추가
    • -d 유저명유저 삭제
    • -M 유저1, ...그룹멤버 세팅
    • -A 유저1, ...관리자 세팅
    • -r비밀번호 삭제
    • -Rnewgrp 명령어와 비밀번호를 통한 그룹 접근 금지 설정
    • -R/etc/gshadow에 !설정이면 -R옵션 적용임
    • -R해당 옵션 취소하려면 그룹 비밀번호 재설정
    • -Q/etc가 아닌 다른 마운트포인트를 가리킴
  • chmod 권한 파일명 — 파일 권한속성 변경
    • -R하위 디렉토리 전체 적용
  • chown 소유자:소유그룹 파일명 — 파일 소유자, 소유그룹 변경
    • -R하위 디렉토리 전체 적용
    • root만 실행가능, quota, 악성코드 우회 등의 이유로
  • usermod — 계정 설정(리눅스)
    • -g — 기본 그룹 변경
    • -G — 보조 그룹 변경
    • -a — 보조 그룹 변경시 기존 그룹 유지
    • -d — 홈 디렉토리 변경
    • -m — -d 사용시 실제 디렉토리 내용을 같이 이동
    • -s — 로그인 쉘 변경
    • -L — 계정 잠금
    • -U — 계정 잠금 해제
    • -U — 계정 잠금 해제
    • -l — 사용자 이름 변경
  • chuser — 계정 설정(유닉스)
  • groupmod— 그룹 설정(리눅스)
    • -a 유저명그룹에 유저 추가

grep

  • -i — 대소문자 구분 안 함
  • -c — 매칭된 라인 수만 숫자로 출력
  • -e — 해당 옵션으로 검색어 다수 지정 가능 -e 패턴 -e 패턴, 하이푼을 패턴에 넣고 싶을 때도 활용
  • -v — 해당 패턴이 없는, 반대로 출력
  • -E 정규식egrep과 동일
  • grep -r "패턴" 경로1 경로 2 ... — 지정 경로내의 모든 파일에서 패턴 찾기
  • -p — 결과를 반환하지 않고 종료상태만 반환
  • -F — 와일드카드, 정규식 적용 X

kill [시그널] [PID]

  • 1 SIGHUPhang up, 프로세스 종료 없이 설정, 파일(config)을 재로드할 때 사용
  • 2 SIGINTinterrupt, 터미널에서 ctrl+c 누를 때 발생하는 신호
  • 3 SIGQUITQuit, Ctrl + \와 같으며 프로세스 종료 후 코어 덤프를 생성함
  • 9 SIGKILLKill, 강제 종료, 프로세스가 신호를 무시할 수 없으며 즉시 사살
  • 15 SIGTERMTerminate, 기본값으로 프로세스에게 종료할 시간을 주는 부드러운 권유
  • 18 SIGCONTContinue, 정지된(stopped) 프로세스를 다시 실행
  • 19 SIGSTOPStop, 프로세스를 일시 정지(Ctrl + Z)와 유사

  • -v패턴에 매칭되지 않는 프로세스들에 시그널 전송
