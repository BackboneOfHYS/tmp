stress-ng option

기본 출력 명령어

  • stress-ng --cpu N --timeout 60s: N이 0이면 사용가능한 최대 코어수
  • stress-ng --cpu-load N: CPU 사용률 N%으로 유지
  • stress-ng --vm N1 --vm-bytes N2 --timeout 60s: N1개의 워커 프로세스, 워커당 N2(ex 1G)만큼 메모리 점유
  • stress-ng --hdd N1 --hdd-opts [wr-seq | wr-rnd] --timeout 30s: N1개의 워커 프로세스 디스크 쓰기 작업
  • streess-ng --hdd N1 --hdd-opts [wr-seq | wr-rnd | rd-seq | rd-rnd]wr-seq->순차 쓰기, wr-rnd->랜덤 위치 쓰기, rd->읽기
  • stress-ng --io N1 --timeout 60s: N1개의 워커가 시스템 콜 부하,

cpu options

  • --cpu [N]: CPU 부하를 생성할 워커(프로세스)의 개수를 지정합니다.
    • 0: 시스템 내의 모든 온라인 CPU 코어 수만큼 워커를 자동으로 생성합니다.
    • N: 사용자가 지정한 N개의 워커를 생성합니다.
  • --cpu-method [method]: CPU가 수행할 연산 알고리즘을 선택하여 부하의 성격을 결정합니다.
    • all: 지원하는 모든 연산 방식을 순차적으로 돌아가며 실행합니다.
    • matrixprod: 행렬 곱셈을 수행합니다. 연산 유닛(ALU)과 캐시 메모리를 동시에 강하게 압박합니다.
    • fft: 고속 푸리에 변환을 수행하며 부동 소수점(FPU) 연산 능력을 집중적으로 테스트합니다.
    • prime: 소수를 찾습니다. 연산뿐만 아니라 CPU의 분기 예측(Branch Prediction) 성능을 확인하는 데 좋습니다.
    • bitops: 비트 단위 연산을 수행하여 논리 회로 계통에 부하를 줍니다.
    • idct: 이산 코사인 변환을 수행하여 멀티미디어 처리 능력을 시뮬레이션합니다.
  • --cpu-load [P]: 각 워커가 CPU를 사용하는 점유율을 P%로 고정합니다.
    • 1-99: 특정 퍼센트만큼만 로드를 유지하여, 시스템이 완전히 멈추지 않은 상태에서의 안정성을 테스트합니다.
    • 100: (기본값) 가용 가능한 모든 자원을 투입하여 최대 부하를 발생시킵니다.
  • --cpu-load-slice [S]: CPU 로드를 조절하기 위해 연산과 휴식을 반복하는 시간 조각(ms)을 설정합니다.
    • 양수(ms): 값이 작을수록 더 촘촘하게 부하와 휴식을 교차하여 미세한 발열 및 전력 변화를 유도합니다.
  • --taskset [list]: 생성된 워커들이 특정 CPU 코어에서만 실행되도록 고정(Affinity)합니다.
    • 0,1: 0번과 1번 코어만 사용하여 부하를 발생시킵니다. 특정 코어의 불량이 의심될 때 유용합니다.
  • --sched [policy]: 워커 프로세스의 커널 스케줄링 정책을 강제로 지정합니다.
    • fifo / rr: 실시간 우선순위 정책으로, 다른 프로세스보다 우선적으로 CPU를 점유하게 만듭니다.
    • batch: CPU를 많이 사용하는 백그라운드 작업 모드로 동작하게 합니다.
    • idle: 시스템에 다른 작업이 없을 때만 실행되는 최하위 우선순위로 설정합니다.
  • --cpu-ops [N]: 각 워커가 수행할 연산 루프의 횟수를 지정합니다.
    • N: 정해진 횟수만큼 연산을 마치면 워커가 자동으로 종료되므로, 정확한 작업량 기반 테스트가 가능합니다.
