RPM 주요 옵션 정리

1. 설치 관련 옵션

-i (install)

새 패키지 설치

rpm -ivh package.rpm
  • -v : verbose (상세 출력)
  • -h : 해시바 표시

-U (upgrade)

기존 버전이 있으면 업그레이드, 없으면 설치

rpm -Uvh package.rpm

-F (freshen)

이미 설치된 경우에만 업그레이드

rpm -Fvh package.rpm

2. 삭제 옵션

-e (erase)

패키지 삭제 (파일명이 아니라 패키지명 사용)

rpm -e nginx

3. 조회(Query) 옵션

-qa

전체 설치 패키지 목록

rpm -qa

-qi

패키지 상세 정보

rpm -qi nginx

-ql

설치 파일 목록

rpm -ql nginx

-qf

특정 파일이 속한 패키지 확인

rpm -qf /usr/sbin/nginx

-qR

의존성 확인

rpm -qR nginx

-qV

파일 변경 여부 확인

rpm -qV nginx

4. 검증 옵션

-V

설치된 파일 무결성 검사

rpm -V nginx

5. 강제 옵션 (주의)

--nodeps

의존성 무시

rpm -ivh --nodeps package.rpm

--replacefiles

파일 충돌 무시

rpm -ivh --replacefiles package.rpm

--force

기존 패키지 강제 덮어쓰기

rpm -ivh --force package.rpm

6. RPM 데이터베이스 관리

메타정보 저장 위치:

/var/lib/rpm/

DB 재구성:

rpm --rebuilddb

7. RPM 파일 보기 명령

내부 파일 보기:

rpm -qpl nginx.rpm

패키지 정보 보기

rpm -qpi nginx.rpm

의존성 보기

rpm -qpR nginx.rpm

면접용 한 줄 정리

RPM은 -i(설치), -U(업그레이드), -e(삭제), -q(조회), -V(검증) 옵션을 중심으로 동작하며, /var/lib/rpm 경로의 데이터베이스에 패키지 메타정보를 저장해 관리한다.
