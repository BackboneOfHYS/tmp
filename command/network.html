Oracle Linux (RHEL 계열) 네트워크 명령어 리스트

“네트워크 확인/트러블슈팅 때 바로 쓰는 명령어”만 간단히 정리

📡 네트워크 상태 / 인터페이스

  • ip addr — 인터페이스 IP 확인 (ifconfig 대체)
  • ip link — NIC up/down 상태, MAC 주소
  • ip link up —
  • nmcli device status — NetworkManager 장치 상태 요약
  • ethtool eth0 — 링크 속도/duplex/케이블 연결 여부

📡 네트워크 상태 변경

  • nmcli con show - 연결 설정 프로파일 확인
  • nmcli device show - 장비 상태 확인
  • nmcli con mod 연결 프로파일 이름 - 기본
  • nmcli con up 연결 프로파일 이름 - 커넥션 on
  • nmcli con down 연결 프로파일 이름 - 커넥션 off
  • ipv4.method manual | auto — 정적 할당 | DHCP 사용
  • ipv4.addresses 192.168.56.10/24 — ipv4 주소 설정
  • ipv4.gateway 192.168.56.1 — 게이트웨이 설정
  • ipv4.dns 8.8.8.8 — 도메인 설정

🧭 라우팅 / 게이트웨이

  • ip route — 라우팅 테이블 (default gateway 확인)
  • nmcli device show eth0 | grep GATEWAY — 특정 인터페이스 게이트웨이

🌐 연결 테스트

  • ping 8.8.8.8 — IP 레벨 통신 확인
  • ping google.com — DNS 포함 통신 확인
  • traceroute 8.8.8.8 — 경로 추적
  • tracepath 8.8.8.8 — traceroute 대체(권한 불필요)

🔍 포트 / 소켓 확인

  • ss -lntup — 열린 TCP/UDP 포트 + 프로세스
    • -t — tcp 포트
    • -t — udp 포트
    • -l — listen 포트
    • -n — DNS/서비스명 변환 안 함(숫자 그대로)
    • -a — listen 포함 모든 소캣
    • -p — pid/프로세스 표시
    • -s — 요약 통계
    • -m — 메모리 사용량 표시
    • -i — tcp 내부 정보 표시
    • -o — 타이머 정보 표시
  • netstat -lntup — 구버전(있으면 참고)

📄 DNS / 이름 해석

  • cat /etc/resolv.conf — DNS 서버 확인
  • getent hosts google.com — 실제 name resolution 결과
  • nslookup google.com — DNS 조회
  • dig google.com — DNS 상세 조회

🔥 방화벽 (firewalld)

  • ip, port 검사
  • firewall-cmd --state — 방화벽 상태
  • firewall-cmd --list-all — 존 기준 전체 설정
  • firewall-cmd --list-ports — 열린 포트
  • firewall-cmd --list-services — 허용 서비스

🔥TCPwrapper

  • 세션, 애플리케이션 계층 작용 직전에? 호스트의 서비스 접속 권한 검사
  • host.allow, deny

⚙️ NetworkManager 설정

  • nmcli connection show — 연결 프로파일 목록
  • nmcli connection show <conn> — 프로파일 상세
  • nmcli connection up <conn> — 연결 활성화
  • nmcli connection down <conn> — 연결 비활성화

🧪 패킷 레벨 (문제 생겼을 때)

  • tcpdump -i eth0 — 인터페이스 패킷 캡처
  • tcpdump -nn port 80 — 포트 필터 + 숫자 출력

🧪 ntp / chrony 설정

  • systemctl start ntpd / chronyd — systemctl 명령으로 ntp / chrony 가동, 중지
  • /etc/chrony.conf — 설정파일 경로
  • /etc/ntp.conf — 설정파일 경로
  • 서버 주소 뒤에 iburst 추가
  • pool poll.ntp.org iburst
  • server time.bora.net iburst
  • 설정 적용하려면 서비스 재시작
  • server는 단일 행만 보고 pool은 여러개 행을 넣을 수 있음
  • server랑 pool 둘 다 설정하면 server먼저 보고 pool은 후보군으로 됨
  • driftfile은 오답노트, 호스트 시계가 얼머나 하루에 빠른가, 느린가
  • makestep은 임계값을 넘은 오차를 바로 점프해서 시간 동기화, 원래는 천천히 맞춤
  • 소프트웨어와 하드웨어 시간 동기화하는 게 rtcsync
  • hardwarestamp는 nic에서 지원해야 되며 ntp/chrony 패킷이 온 정확한 시점을 찍어서 cpu가 안 바쁠 때? 전달
  • minsources는 pool에서 다수결투표의 최소 개수를 지정
  • maxsourrces는 pool에서 다수결투표의 최대 개수를 지정
  • allow는 내 서버를 ntp 서버로 변신시킴
  • startum은 시간 신뢰 등급이며 ntp가 끊기면 기본으로는 다른 장비들에게 시간 정보 주지 않음
  • 하지만 local startum 10을 설정하면 호스트를 바로보는 클라이언트들이 10등급의 대장서버로 인식함
  • /etc/chrony.keys번호/알고리즘/비밀번호, 번호는 chrony.conf에서 사용, 비밀번호는 server-client에서 동일하게 가져야함
  • 윤초(지구의 자전시간과 원자시계 오차 맞추기 위해 1초 더하거나 빼는 것)
  • leapsectz는 윤초를 처리할 때 시스템 타임존 데이터베이스(/usr/share/zoneinfo)을 참고하라는 명령어
  • 최근에는 leap smearing을 더 선호함(1초를 바로 더하지 않고 천천히 더함)
  • log 선택 가능한 정보 리스트
    • measurements각 NTP 소스로부터 받은 원시 측정값(가장 세밀함)
    • statistics각 소스에 대해 수행된 회귀 분석 통계(안정성 판단용)
    • tracking시스템 시계의 이득/손실 속도 및 보정된 오차 기록
    • refid각 소스의 참조 ID(Refid) 정보
    • rtc시스템의 하드웨어 시계(RTC)에 대한 오차 및 보정 기록
    • rawmeasurements필터링되지 않은 순수 네트워크 패킷 타임스탬프(디버깅용)
    • subseconds(일부버전) 소수점 이하의 아주 정밀한 시간 단위 정보
