signal, status code concept

signal이란?

  • OS가 해당 프로세스에게 정해진 번호의 신호를 보냄
  • signal을 받은 프로세스는 아래 세 가지 중 한 가지 행동을 함
    • 기본 동작OS가 정해준 기본 설정대로 행동
    • 무시시그널을 못 본 척하고 자기 할 일을 계속 함
    • 가로채기시그널이 오면 미리준비한 코드 실행(catch, trap)

언제 사용하는지

  • 사용자 개입
  • 시스템/커널 경고
  • 관리자 서버 제어
  • 프로세스끼리 통신

signal 종류

  • SIGHUP (1) 터미널 종료 (Hangup)
  • SIGINT (2) 인터럽트 (Ctrl+C)
  • SIGQUIT (3) 터미널 종료 (Ctrl+)
  • SIGILL (4) 잘못된 명령어
  • SIGTRAP (5) 디버그 트랩
  • SIGABRT (6) 프로세스 중단 (Abort)
  • SIGBUS (7) 버스 에러 (잘못된 메모리 접근)
  • SIGFPE (8) 산술 연산 오류 (0으로 나누기 등)
  • SIGKILL (9) 강제 종료 (가로채기/무시 불가)
  • SIGUSR1 (10) 사용자 정의 시그널 1
  • SIGSEGV (11) 세그먼테이션 폴트 (메모리 침범)
  • SIGUSR2 (12) 사용자 정의 시그널 2
  • SIGPIPE (13) 파이프 끊김 (데이터 받을 프로세스가 없음)
  • SIGALRM (14) 알람 타이머 종료
  • SIGTERM (15) 종료 요청 (Terminate)
  • SIGSTKFLT (16) 스택 결함
  • SIGCHLD (17) 자식 프로세스 상태 변경 (종료 등)
  • SIGCONT (18) 멈춘 프로세스 재개
  • SIGSTOP (19) 일시 정지 (가로채기/무시 불가)
  • SIGTSTP (20) 사용자의 일시 정지 (Ctrl+Z)
  • SIGTTIN (21) 백그라운드 프로세스가 입력을 읽으려 함
  • SIGTTOU (22) 백그라운드 프로세스가 출력을 하려 함
  • SIGURG (23) 소켓의 긴급 데이터(Out-of-band) 도착
  • SIGXCPU (24) CPU 시간 제한 초과
  • SIGXFSZ (25) 파일 크기 제한 초과
  • SIGVTALRM (26) 가상 타이머 종료
  • SIGPROF (27) 프로파일링 타이머 종료
  • SIGWINCH (28) 터미널 윈도우 크기 변경
  • SIGIO (29) 입출력 가능 상태
  • SIGPWR (30) 전원 부족/장애 발생
  • SIGSYS (31) 잘못된 시스템 콜 호출

종료 상태 코드

  • success (0)성공적 실행
  • general error (1)일반적인 에러 (가장 흔함)
  • command invoked cannot execute (126)명령어를 찾았으나 실행 권한 없음
  • command not found (127)명령어 자체가 존재하지 않음($PATH에 없음)
  • fatal error signal "n"(128+n)시그널 n에 의해 종료됨
