Linux cron 정리

crontab 옵션

  • -e예약된 작업 목록 편집
  • -l현재 설정된 작업 목록을 확인
  • -r현재 설정된 모든 작업을 삭제

crontab 설정

  • 분 시 일 월 요일 [명령 또는 스크립트]
  • /etc의 cron의 경우 분 시 일 월 요일 [사용자 실행 권한] [명령 또는 스크립트]로 구성됨

crontab 설정파일

  • /var/spool/cron/crontabs일반 사용자가 -e 명령어로 생성한 파일들이 사용자 이름 별로 저장
  • crontab -e 명령으로 사용해야 시스템이 즉시 감지함
  • /etc/crontab시스템 전역 크론 설정
  • /etc/cron.d/설치된 소프트웨어나 패키지들의 크론 설정
  • /etc/cron.hourly, daily, weekly, monthly정해진 주기에 맞춰 실행되며 /etc/crontab이나 anacron에 실행 정의가 되어있음
  • cron.allow, deny크론 접근 설정이며 allow 파일이 있으면 해당 파일에 기재된 유저만 cron 설정 가능

anacrontab

  • /etc/anacrontabanacron 설정파일로 실행되지 않은(부팅, 기타 등의 사유로) cron을 실행
  • period delay job-identifier commandanacron 설정값, 각각 작업주기, 부팅 후 실행 대기시간, 작업고유이름, 명령어
  • anacron은 부팅시, 이후엔 cron에 의해 실행됨
