DNF 주요 옵션(명령) 정리

1) 설치 / 삭제

설치 install

dnf install nginx

여러 개 설치

dnf install nginx httpd git

삭제 remove

dnf remove nginx

재설치 reinstall

dnf reinstall nginx

자동 제거(불필요 의존성 정리) autoremove

dnf autoremove

2) 검색 / 조회

검색 search

dnf search nginx

패키지 정보 info

dnf info nginx

목록 조회 list

dnf list installed
dnf list available
dnf list all

파일을 제공하는 패키지 찾기 provides

dnf provides /usr/sbin/nginx

※ rpm -qf에 해당하는 상위 개념(Repo까지 포함해서 검색 가능)

3) 업데이트 관련

전체 업데이트 update / upgrade

dnf update
dnf upgrade

업데이트 가능 목록 확인 check-update

dnf check-update

특정 패키지만 업데이트

dnf update nginx

4) Repo(저장소) 관련

활성 Repo 확인 repolist

dnf repolist

전체 Repo 확인

dnf repolist all

캐시 삭제 clean

dnf clean all

특정 Repo만 사용 --enablerepo / --disablerepo

dnf --enablerepo=repo이름 install nginx
dnf --disablerepo=* --enablerepo=myrepo install nginx

5) 의존성 / 문제 해결

의존성 확인 deplist

dnf deplist nginx

시스템 점검 check

dnf check

6) 다운로드만 (설치 안 함)

dnf download nginx

7) 자주 쓰는 옵션 조합

자동 yes -y

dnf -y install nginx

특정 버전 설치(가능한 경우)

dnf install nginx-1.20.1

Repo 캐시 재생성(문제 해결)

dnf clean all
dnf makecache

8) dnf 관련 파일 경로

  • /etc/yum.repos.d/
    → 개별 .repo 저장소 설정 파일들이 위치하는 디렉터리
  • /etc/dnf/
    → dnf 패키지 매니저의 전역 설정 파일이 위치하는 디렉터리
  • /etc/pki/rpm-gpg/
    → 패키지 무결성 검증에 사용하는 GPG 키 파일 저장 위치
  • /var/cache/dnf/
    → dnf가 다운로드한 메타데이터 및 패키지 캐시 저장 위치
  • /var/cache/yum/
    → yum이 다운로드한 메타데이터 및 패키지 캐시 저장 위치

DNF vs RPM 차이 핵심

DNF RPM
Repo 기반 설치/업데이트 로컬 .rpm 파일 중심 설치/조회/삭제
의존성 자동 해결 의존성 수동(강제 옵션 사용 시 위험)
상위 관리자(실무 기본) 저수준 관리자(기초/검증/특수 상황)

면접용 한 줄

DNF는 RPM을 기반으로 동작하는 상위 패키지 관리자로, install, remove, update, search, repolist 등의 명령을 통해 저장소 기반 설치와 의존성 자동 처리를 수행한다.
