Oracle Linux (RHEL 계열) 파일/스토리지 명령어 리스트

기본 서버 점검에서 사용하는 명령어만 간단히 정리

참고지식

  • VFS — 가상 파일 시스템, 파티션뿐만 아닌 모든 것을 파일로 다룸

기본 명령어 흐름

  • lsblk — 디스크 구조 전체 트리 확인
  • blkid — UUID 및 파일시스템 타입 확인
  • fdisk [장치명] — 파티션 생성 및 확인
  • pvcreate — 물리 볼륨 생성
  • vgcreate — 볼륨 그룹 생성
  • lvcreate — 논리 볼륨 생성
  • mkfs.[파일시스템] [디바이스경로] — 파일시스템 생성
  • mount / findmnt — 마운트 상태 확인
  • df -h — 파일시스템 사용률 점검
  • df -P — 출력을 무조건 한 줄로 고정(장치명이 길면 줄바꿈하는 것 제거)
  • du -h — 특정 파일 시스템 내부 점검
  • lsof — 열려 있는 모든 파일 확인
  • df -i — inode 사용량 확인
  • genisoimage — iso 이미지 생성
  • mount -o loop 파일명.iso 마운트디렉토리 — iso파일 마운트

VFS 기반 가상 파일시스템

  • proc — 커널과 프로세스 정보 제공, /proc
  • sysfs — 커널 장치/드라이버 정보, 하드웨어 제어용 인터페이스, /sys
  • tmpfs — RAM 기반 파일시스템, 재부팅시 소멸, 빠름, /dev/shm, /run
  • devtmpfs — 장치 파일 저장, 커널이 자동 생성, /dev
  • cgroup, cgroup2 — 프로세스 자원 제어(CPU, MEM), 컨테이너 환경에서 매우 중요

네트워크 파일시스템

  • nfs — NAS 기반, 리눅스 간 공유
  • cifs — Samba 기반, 윈도우 파일 공유

특수목적 파일시스템

  • overlay — 여러 파일시스템을 겹쳐서 하나처럼 사용, 컨테이너에서 사용
  • squashfs — 읽기 전용 압축 파일시스템
  • autofs — 자동 마운트 시스템, 접근 시 자동으로 마운트

pseudo filesystem

  • mqueue — POSIX 메시지 큐
  • hugetlbfs — HugePage 메모리
  • debugfs — 커널 디버깅
  • tracefs — 커널 트레이싱
  • securityfs — 보안 모듈 인터페이스

fidsk 옵션 정리

  • step.1 p — 파티션 테이블 정보
  • step.1 l — 지원하는 파티션 타입
  • step.1 v — 파티션 테이블 오류 검증
  • step.1 n — 파티션 생성
  • step.1 d — 파티션 삭제
  • MBR 디스크는 4개만 가능하며 기본 파티션이 이 4개이고, 나머지는 e로 해야함
  • MBR(Master Boot Record)로 최대 2TB까지만
  • GPT(GUID Partition Table)로 최대 18EB까지 지원
  • OS 커널은 MBR과 GPT 모두 관련 라이브러리를 대게 가지고 있으며
  • fdisk 장치경로 후 o -> MBR, g -> GPT
  • fdisk 장치경로 후 p -> 정보 출력 -> disklable type 확인(dos or gpt)
  • step.2 p — 파티션 테이블 정보
  • step.2 e — 지원하는 파티션 타입

du 옵션 정리

  • 물리적 사용량, ls -l에서의 크기에서 디렉터리 크기는 하위 파일 미포함임
  • -s — 각 항목 합계만 출력
  • -h — 사람이 읽기 좋은 단위
  • -k — KB 단위
  • -g — GB 단위
  • --max-depth=n — n단계 디렉터리만
  • --exclude — 특정 파일시스템 제외

lsvg, vgs 옵션 정리

  • null — VG 목록
  • VG이름 — VG 상세 정보
  • -l — VG 안의 LV 목록
  • -p — VG 안의 PV 목록
  • -o — vgs는 컬럼 선택, lsvg는 활성화 상태 vg 출력
  • -i — lsvg는 표준입출력 파이프라인으로 사용함, lsvg -o | lsvg -i와 같이
  • stale — 디스크가 2개 이상 미러돼 있을 때 한쪽에 쓰기가 제대로 반영되지 않은 상태, lsvg -l에서 발생
  • miss — OS가 해당 PV를 찾지 못하는 상태, lsvg -p에서 발생

lvm 관련 명령 정리

  • pvcreate [장치명] — pv 생성
  • pvremove — pv 삭제
  • vgcreate [vg명] [pv명] [pv명] ... — vg 생성
  • vgextend [vg명] [장치명] — vg 확장
  • vgremove — vg 삭제
  • lvcreate -L [용량] -n [lv명] [그룹명] — lv 생성
  • lvextend -L [용량] [lv명] — lv 확장
  • lvreduce -L [용량] [lv명] — lv 축소
  • lvextend, lvreduce -r — 파일 시스템 확장 자동 수행
  • lvextend, lvreduce -L — 사람이 보기 쉬운 단위
  • lvremove — lv 삭제

파일시스템 검사 및 복구

  • fsck [옵션] [장치명] — 파일 시스템 점검 및 복구
    • 디스크 I/O 캐시로 인해 오류발생 가능하므로 umount 필수
    • -a — 오류 발견 시 묻지 않고 자동 복구
    • -r — 오류 복구 시 사용자에게 확인 요청
    • -y — 모든 질문 yes
    • -n — 수정하지 않고 점검만 수행
    • -f — 강제 점검 진행
    • -t [타입] — 점검 대상 파일시스템 지정
    • -V — 상세 출력
  • e2fsck — ext2/3/4 파일시스템 타겟 체크 도구
  • badblocks [옵션] [장치명] — 물리적 손상으로 데이터 rw 불가한 배드 섹터 검색 도구
    • 물리적 배드 섹터 - 물리적 파손
    • 논리적 배드 섹터 - 기록 오류로 ECC값 오류
    • 리매핑 - 배드섹터를 봉인하고 예비 섹터를 할당하는 것
    • -v — 상세 출력
    • -o [파일명] — 결과 파일 저장
    • -n — 데이터를 유지하면서 검사
    • -w — 데이터 삭제, 특정 패턴으로 검사
    • -s — 검사 진행률을 %를 써서 출력
  • dump — 백업도구
  • restore — 복구도구
  • dumpe2fs — ext2/3/4/ 설정정보 조회

쿼터 관리

  • quotacheck — 사용량 체크 및 쿼타 기록 갱신
  • qutaon/off — 쿼타 기능 활성화
  • edquota — 쿼타 설정 편집
  • quota — 쿼타 설정 확인
  • repquota — 보고서 형태로 출력

RAID 컨트롤러

  • lspci — pci 포트 연결 확인
  • lsmod — 드라이버 로드 여부 확인
  • storcli — Broadcom / LSI 벤더사 명령어
  • ssacli — HP(HPE) 벤더사 명령어
  • perccli — DELL 벤더사 명령어

FC HBA

  • cat /sys/class/fc_host/host*/port_name — WWN 확인 후 스토리지에 등록해야 디스크 보임
