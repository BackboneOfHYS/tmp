Oracle Linux (RHEL 계열) 파일/스토리지 명령어 리스트

기본 서버 점검에서 사용하는 명령어만 간단히 정리

참고지식

  • VFS — 가상 파일 시스템, 파티션뿐만 아닌 모든 것을 파일로 다룸

기본 명령어 흐름

  • lsblk — 디스크 구조 전체 트리 확인
  • blkid — UUID 및 파일시스템 타입 확인
  • mount / findmnt — 마운트 상태 확인
  • df -h — 파일시스템 사용률 점검
  • du -h — 특정 파일 시스템 내부 점검
  • lsof — 열려 있는 모든 파일 확인
  • df -i — inode 사용량 확인

VFS 기반 가상 파일시스템

  • proc — 커널과 프로세스 정보 제공, /proc
  • sysfs — 커널 장치/드라이버 정보, 하드웨어 제어용 인터페이스, /sys
  • tmpfs — RAM 기반 파일시스템, 재부팅시 소멸, 빠름, /dev/shm, /run
  • devtmpfs — 장치 파일 저장, 커널이 자동 생성, /dev
  • cgroup, cgroup2 — 프로세스 자원 제어(CPU, MEM), 컨테이너 환경에서 매우 중요

네트워크 파일시스템

  • nfs — NAS 기반, 리눅스 간 공유
  • cifs — Samba 기반, 윈도우 파일 공유

특수목적 파일시스템

  • overlay — 여러 파일시스템을 겹쳐서 하나처럼 사용, 컨테이너에서 사용
  • squashfs — 읽기 전용 압축 파일시스템
  • autofs — 자동 마운트 시스템, 접근 시 자동으로 마운트

pseudo filesystem

  • mqueue — POSIX 메시지 큐
  • hugetlbfs — HugePage 메모리
  • debugfs — 커널 디버깅
  • tracefs — 커널 트레이싱
  • securityfs — 보안 모듈 인터페이스

du 옵션 정리

  • 물리적 사용량, ls -l에서의 크기에서 디렉터리 크기는 하위 파일 미포함임
  • -s — 각 항목 합계만 출력
  • -h — 사람이 읽기 좋은 단위
  • -k — KB 단위
  • -g — GB 단위
  • --max-depth=n — n단계 디렉터리만
  • --exclude — 특정 파일시스템 제외

lsvg, vgs 옵션 정리

  • null — VG 목록
  • VG이름 — VG 상세 정보
  • -l — VG 안의 LV 목록
  • -p — VG 안의 PV 목록
  • -o — vgs는 컬럼 선택, lsvg는 활성화 상태 vg 출력
  • -i — lsvg는 표준입출력 파이프라인으로 사용함, lsvg -o | lsvg -i와 같이
  • stale — 디스크가 2개 이상 미러돼 있을 때 한쪽에 쓰기가 제대로 반영되지 않은 상태, lsvg -l에서 발생
  • miss — OS가 해당 PV를 찾지 못하는 상태, lsvg -p에서 발생
