boot concept

boot 과정에 대하여

  • POST(Power On Self Test) 과정 후 BIOS or UEFI 부팅 진행
  • BIOS디스크 인식을 USB, SSD, HDD 순으로 하며 이 순으로 부트로더를 찾음(MBR 섹터)
  • UEFI모든 디스크를 보고 부트로더 엔트리를 목록화하여 사용자가 선택함(EFI 시스템 파티션의 .efi 파일)
  • 부트로더는 GRUB가 대표적이지만 다른 것도 있음
  • Windows Boot ManagerWindows
  • AppleApple 자체 부팅 시스템
  • SysLinuxGRUB의 경량버전
  • SysLinux, isoLinux설치 USB에서 동작
  • 부트로더는 여러 커널을 엔트리 목록으로 하여 사용자가 선택할 수 있도록 함

boot time에 대하여

  • 서버급 PC는 부팅 시 POST 과정이 김
  • Memory Traing -> 메모리 크기가 클 수록 증가, ECC로 더 증가
  • RAID Controller -> RAID카드 별로 컨트롤러(별도 BIOS)로드 및 드라이브 상태 확인
  • 스토리지가 온보드(내장)이면 시간 단축, 외장이면 시간 증가
  • Remote Management(IPMI) -> 서버 관리 모듈 초기화, 본체 부팅 전에 부팅
  • PXE BOOT?
  • 고성능 NIC, GPU -> 펌웨어 로드 시간 증가
  • Detail check, Quick Boot로 조절 가능
  • BIOS vs UEFI 모드 선택
