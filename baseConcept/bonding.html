bonding concept

bonding 기본

  • 본딩은 물리 nic를 이중화하기 위함
  • 본딩이 아니라 nic가 여러개라면 하나의 nic에 개별설정이 가능
  • 하지만 여러 nic를 하나의 본딩으로 묶으면 본딩 설정만 가능
  • 본딩시 스위치<->서버 간 mac 주소 꼬임이 발생할 수 있음
  • /proc/net/bonding/$con-name 으로 bonding 옵션 출력
  • teamd는 본딩보다 복잡한 설정을 활용할 때 사용
  • nmcli con add type bond con-name bond0 ifname bond0 mode active-backup -> mode 1번
  • nmcli con add type bond con-name bond0 ifname bond0 mode 802.3ad -> mode 4번
  • nmcli con del bond0(bond0=port1)
  • nmcli con show bond0
  • nmcli con add type bond-slave con-name bond-port0 ifname $장치명 master $본딩명
  • nmcli con mod bond0 +bond.option "???"
  • nmcli con mod bond0 +bond.option "miimon=100"

actvie-backup

  • 스위치 몰래 선을 바꾸는 방식
  • nmcli con mod bond0 +bond.option "primary=장치명"으로 최우선 nic 설정 가능
  • 해당 모드의 경우 primary nic의 fail over 발생 시 nic mac 꼬임이 발생함
  • 이를 해결하기 위한 방법
    • nmcli con mod bond0 +bond.option "fail_over_mac=active로 맥주소 전환
    • 스위치 설정 or 무차별 모드 on or 주소 변조 차단 해제

LACP(Link Aggregation Control Protoco)

  • 여러 개의 nic or 포트를 하나의 논리 nic, or 포트로 묶어 대역폭 확장, 장애 복구 제공, IEEE 802 3ad 표준 프로토콜
  • 스위치는 MAC 주소를 Port-Channel로 묶어서 관리하므로 해당 채널에 속한 맥주소는 통과시킴
  • Active - 상대방이 누구든 먼저 본딩을 제안하는 성격, 서버는 기본 active
  • passive - 상대방의 요청에만 본딩을 구성하는 성격
  • 보통 스위치-서버 둘 다 active여야 재기동 등에도 바로 붙음
  • nmcli con mod bond0 +bond.option "mode=4,lacp_rate=fast"
  • mcli connection add type team con-name team0 ifname team0 config '{"runner": {"name": "lacp", "active": true}}'
