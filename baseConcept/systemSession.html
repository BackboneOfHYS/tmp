Linux system session

what is system session?

  • 사용자 세션 / 시스템(데몬) 세션 으로 구분됨
  • 위 둘의 가장 큰 차이는 표준입출력(제어터미널) stdin, stdout, stderr의 유무
  • 사용자 세션의 세션리더 프로세는 대부분 쉘
  • 시스템(데몬) 세션의 세션리더는 데몬 프로세스 그 자체
  • systemctl로 start하면 systemctl이 D-Bus 통신 채널로 실행 요청
  • systemd(시스템 데몬, pid 1)이 fork(), setsid()명령(또는 systemd의 Cgroup 안에서)으로 새로운 데몬 세션 생성
  • 제어터미널은 입출력 통로
  • 세션은 터미널과 연결된 모든 프로세스를 하나로 묶는 관리 단위
  • 쉘은 세션이 시작될 때 가장 먼저 실행, 세션 내 다른 프로세스 실행
  • 쉘은 세션 설정에 접근할 수 있음
  • 데몬 프로세스는 쉘이 없음

how make session?

  • 부팅 후 최초 프로세스 생성
    • 0번(idle/swapper) - 커널, 프로세스 스케줄링 담당
    • 1번(init/systemd) - 모든 프로세스의 조상, 시스템 서비스와 데몬 관리
    • 2번(kthreadd) - 커널 스레드 관리
  • systemd가 getty 실행
    • 터미널 선점 및 초기화
    • 키보드 1개 -> 가상 스위치 (기본 tty1, ctrl + alt + Fn으로 변경 가능) -> real 터미널(/dev/ttyn) -> 커널의 가상 터미널(실제 입출력 담당)
    • 표준입출력(제어터미널) 연결
    • 사용자 이름 대기
  • getty가 exec 명령으로 login 프로세스 실행
  • 인증 과정
  • 로그인 성공 후 로그인 프로세스 동작
    • 로그인 프로세스 권한 변경(root->user)
    • setsid() 호출, 세션 리더 등록
    • 환경 변수 로드
  • login이 exec명령으로 사용자 설정의 로그인 쉘 실행

what have session?

  • 환경변수
  • 프로세스 그룹 관리
  • 리소스 제한
  • umask
  • tty 설정(제어터미널과의 통신규칙)
  • namespace & cgroups
  • 쉘 옵션 플래그는 초기에는 쉘이 세션을 훑고 스스로 결정
  • 이후에는 사용자가 관리
