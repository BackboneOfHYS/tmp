network base concept

packet 구조

  • 패킷은 송수신의 ip, mac 등 각 계층별 헤더를 모두 담고 있다
  • 패킷을 보고 각 계층별 장비는 역할에 따라 보는 헤더가 다르다

라우팅 구조

  • 가상 포트란 물리포트에 VLAN을 부여한 것으로 패킷에 태그를 붙여서 통신함
  • 트렁크 포트란 VLAN을 구성하는 스위치와 라우터를 연결하며 이 방식을 Router on a Stick이라고 함
  • VLAN이란 하나의 물리 스위치나 물리 포트(라우터)를 논리적으로 구분한 것, 물리 스위치는 포트마다 VLAN을 설정해줌
  • 라우터는 관리자가 물리(가상) 포트에 네트워크 대역을 설정해주어야 함
  • 라우터는 목적지 IP와 가장 길게 일치하는 네트워크 대역으로 라우팅해줌
  • 라우팅 테이블이란 관리자가 물리(가상) 포트에 입력한 것을 기본으로 다른 라우터들과 통신해서 통신맵 구성
  • 라우터는 NAT/PAT 수행으로 망의 검문소 역할
  • L3 스위치는 빠른 스위칭 역할
  • VIP는 클러스터링과 로드밸런서로 구분됨
  • 로드밸런서는 L4 스위치로 작동되며 HA 구성에 Active-Active에 해당함
  • 로드밸런서는 L3보다 뒤에 위치함, 밸런싱 역할이기 때문에
  • L7스위치는 L4를 대체하거나 L4보다 뒤에 위치함
  • 클러스터링은 백그라운드 프로세스로 구동되며 구성서버에 개별적으로 설치, 이후 서버끼리 통신으로 VIP 구성, Active-Stanby에 해당
  • 전후방 IP에서 VIP는 전방, 후방 모두 존재할 수 있다.
  • 전방은 외부에서 SP(Service Provider)를 바라보는 주소, 후방은 내부 관리 주소
  • 따라서 DMZ의 전방은 공인 IP로 볼 수 있음
  • 게이트웨이는 자신이 속한 네트워크의 대표 주소로 다른 네트워크에 보내는 걸 대행해줌

네트워크 토폴로지

  • 망이 다르면 라우터를 한 번 거쳐야 됨
  • 고로 여기서 라우팅 설정과 네트워크 장비가 필요하고
  • 망마다 방화벽, LB, WAF, IDS, IPS, NAC, L7 존재로 각 장비마다 포트 설정 필요
  • 망 유형(스타형, 버스형, 트리형)등 고려할 필요도 있음
  • 백본망이 다 이어주는 망이락 생각하면 편함
  • 백본망에 각 망의 게이트웨이나 라우터가 붙어있음

패킷 처리 프레임워크 iptables

  • 라우팅 경로 지정이 아닌 어떻게 처리할 지에 대한 것
  • packet in -> prerouting -> 라우팅 결정 -> forward / input -> post routing -> packet out
  • echo 1 > /proc/sys/net/ipv4/ip_forward - IP 포워딩 켜기
  • 필터링(방화벽), filter
  • NAT(주소 변환), nat
  • 포워딩(라우팅 제어)
  • MANGLE - 패킷 속성 수정

NAT(주소 변환), nat

  • IP / Port 변경
  • NAT(SNAT, DNAT, MASQUERADE)
  • PAT는 SNAT, DNAT 둘 다 발생할 수 있음
  • 처음 패킷에만 적용되며 이후 conntrack(상태 추적) 처리됨
  • PREROUTING - 전방, 들어오자마자, DNAT
  • POSTROUTING - 후방, 나가기 직전, SNAT, MASQUERADE
  • OUTPUT - 로컬 NAT(로컬 서버에서 직접 만든 패킷)
  • MASQUERADE - 동적 SNAT, 공인 IP 변동에 자동 대응
  • SNAT - 정적 SNAT, 출발지 고정 서버
  • DNAT - 포트포워딩, 목적지IP/포트를 내부 서버로 변경
  • iptables -t nat -A [체인] 조건 -j [타겟]
    • -t nat - NAT 테이블
    • -A - 규칙 추가
    • -D - 규칙 삭제
    • -L - 조회
    • -n - 숫자로 출력
    • -v - 상세 출력
    • -p tpc/udp - 프로토콜
    • -s - 출발지 IP
    • -d - 목적지 IP
    • --sport - 출발지 포트
    • --dport - 목적지 포트
    • -i - 입력 인터페이스
    • -o - 출력 인터페이스
    • -j MASQUERADE/SNAT/DNAT - 타겟

포워딩(라우팅 제어), mangle?

  • 라우팅은 길 안내, 포워딩은 실제 패킷을 보내는 행위

filter

  • INPUT - 내 서버로 들어온 패킷
  • OUTPUT - 로컬에서 생성, 로컬 NAT
  • FORWARD - 남의 패킷 중계

MANGLE

  • 수행 위치는 다양함, 각 목적별로 수행 기대 위치가 다를 수 있기 때문
  • TTL 변경 - 패킷 생존 횟수 조작
  • TOS /DSCP (QoS) - 트래픽 우선순위 설정
  • MARK - 패킷에 라벨붙임, 이후 라우팅에서 사용
