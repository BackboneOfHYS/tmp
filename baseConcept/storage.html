block group이란?

  • inode와 data를 가까이 배치하기위함 -> seek time 감소
  • ex) [inodes][datas] -> [inodes_1][datas_1][inodes_2][datas_2][inodes_3][datas_3]
  • block group마다 각자만의 data block을 갖는 것이 아님
  • extent 방식에서 실제 블록 위치 계산은 group_num = block_number / blocks_per_group으로
  • 블록 색인을 그룹별 블록 수로 나눈 값으로 접근함

inode란?

  • inode란? 파일권한, 크기, 소유자, 타임스탬프, 데이터 위치를 저장하나 파일명은 directory entry에 저장함
  • inode 메타데이터에 mode 항목에 파일타입이 정해짐
  • inode가 가리키는 extent가 많으면 일반 리스트가 아닌 B-Tree로 확장

data block에 실제 데이터 저장

  • extent 등장으로 기존 block pointer 방식에서 block 1000~2000(extent 범위 가리킴)와 같이 변경됨
  • -> 크기나 내부구조가 아닌 블록 관리 구조의 차이
